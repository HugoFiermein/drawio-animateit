# AnimateIT! для draw.io

AnimateIT! помогает отслеживать выбранные соединители в draw.io / diagrams.net. Когда вы выбираете линию, она получает яркую анимацию потока, временно поднимается над пересекающимися линиями, а после снятия выделения возвращается к исходному стилю и порядку слоя.

Плагин полезен для сетевых схем, стоек, инфраструктурных диаграмм и любых схем, где много пересекающихся соединителей.

## Возможности

- Анимация потока на выбранных соединителях
- Ярко-зелёная подсветка и увеличенная толщина линии
- Временный подъём выбранной линии на передний план
- Восстановление исходного стиля и порядка слоя после снятия выделения
- Включение и выключение через меню draw.io **Дополнительно** или `Ctrl + Alt + I`
- Варианты для desktop-приложения и браузера

## Установка в браузере через Tampermonkey

Используйте этот вариант для официального веб-редактора [app.diagrams.net](https://app.diagrams.net/). Userscript — самый практичный способ для браузера, потому что обычная загрузка внешних плагинов может блокироваться политикой безопасности сайта.

1. Установите Tampermonkey.
2. В Chrome откройте **Расширения -> Tampermonkey -> Сведения** и включите **Разрешить пользовательские скрипты**. Без этой галочки скрипт может быть установлен, но не будет запускаться.
3. Откройте Tampermonkey и выберите **Создать новый скрипт**.
4. Вставьте полное содержимое `AnimateIT.user.js`.
5. Сохраните скрипт через `Ctrl + S`.
6. Откройте или полностью перезагрузите [app.diagrams.net](https://app.diagrams.net/) через `Ctrl + F5`.

Когда всё работает, в консоли браузера появляются строки:

```text
[AnimateIT] Userscript started...
[AnimateIT] Draw.loadPlugin found...
[AnimateIT] Ready
```

Также можно выполнить в DevTools:

```js
window.__AnimateITStatus
```

`ready: true` означает, что userscript успешно подключился к draw.io.

## Установка в desktop-версии

Начиная с draw.io Desktop 19.0.3 внешние плагины отключены по умолчанию, поэтому приложение нужно запускать с флагом `--enable-plugins`.

1. Скачайте `AnimateIT.js` и сохраните его в постоянное место, например:

   ```text
   C:\Users\YourName\drawio-plugins\AnimateIT.js
   ```

2. Откройте свойства ярлыка draw.io и добавьте флаг в поле запуска:

   ```cmd
   "C:\Program Files\draw.io\draw.io.exe" --enable-plugins
   ```

3. Откройте draw.io через этот ярлык.
4. Перейдите в **Дополнительно -> Плагины...**.
5. Нажмите **Добавить**, выберите `AnimateIT.js`, затем нажмите **OK** и **Применить**.
6. Перезапустите draw.io через тот же ярлык.

После перезапуска пункт **Дополнительно -> AnimateIT!** появится в меню.

## Self-hosted / Docker

Если вы поднимаете diagrams.net самостоятельно, например через Docker, вы контролируете HTTP-заголовки и источник плагинов. В таком варианте можно использовать `AnimateIT.js` как обычный draw.io-плагин.

Рекомендуемая схема:

1. Разместите `AnimateIT.js` на том же домене, что и ваш diagrams.net, или настройте CSP так, чтобы URL плагина был разрешён.
2. Откройте ваш self-hosted diagrams.net.
3. Добавьте плагин через **Дополнительно -> Плагины...**.
4. Перезагрузите редактор.

## Использование

Выберите любой соединитель. AnimateIT! подсветит его и поднимет над пересекающимися линиями. Когда вы снимете выделение, плагин восстановит исходный стиль и порядок слоя.

Используйте `Ctrl + Alt + I` или **Дополнительно -> AnimateIT!**, чтобы временно отключить или снова включить плагин без перезапуска draw.io.

## Совместимость

| Платформа | Статус | Комментарий |
|---|---|---|
| app.diagrams.net / draw.io в Chrome | Поддерживается через Tampermonkey | Нужна галочка **Разрешить пользовательские скрипты** для Tampermonkey |
| app.diagrams.net / draw.io в других браузерах | Поддерживается через userscript-менеджер | Расширению должен быть разрешён доступ к сайту |
| draw.io Desktop для Windows | Поддерживается | Нужен запуск с `--enable-plugins` |
| Self-hosted diagrams.net / Docker | Поддерживается | Разместите плагин там, где его разрешает CSP |
| draw.io для Confluence / Jira | Не поддерживается | Эти окружения не позволяют подключать произвольные плагины |

## Диагностика

Если версия для Tampermonkey не запускается:

- Проверьте, что установленный скрипт начинается с `// @version      1.3.0`.
- Откройте DevTools -> Console и отфильтруйте сообщения по `AnimateIT`.
- Выполните `window.__AnimateITStatus` в консоли, чтобы увидеть последний статус запуска.
- Если сообщений `[AnimateIT]` нет и `window.__AnimateITStatus` возвращает `undefined`, Tampermonkey не запустил скрипт. В Chrome включите **Разрешить пользовательские скрипты** для Tampermonkey и проверьте доступ расширения к `app.diagrams.net`.
- Если виден timeout ожидания `Draw.loadPlugin`, полностью перезагрузите страницу после загрузки редактора.
- Если пункт меню появился, но выделение не работает, проверьте, что выбран именно соединитель, а не фигура.

## Для сопровождающих

`AnimateIT.user.js` намеренно использует `@grant unsafeWindow`, чтобы получить доступ к глобальным объектам draw.io: `Draw`, `mxEvent` и `mxUtils`. Предыдущий вариант с инжекцией через `<script>` удалён, потому что в Chrome он оказался более хрупким и мог падать ещё до установки диагностического статуса.
